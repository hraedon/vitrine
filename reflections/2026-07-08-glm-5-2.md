## On the project

vitrine is a genuinely interesting project — a static-site generator where the data model IS the exhibit, and the mechanical provenance gate (`vitrine check`) is the core innovation. The fact model is well-designed: the Tier/Panel/Basis closed sets with `assert_never()` dispatch, the `amount_minor` integer-cents approach, and the "render the gap" ethos all work together to make the data files self-documenting. The AGENTS.md is one of the best project orientation docs I've seen — it tells you exactly what the project is, what the rules are, and where the bodies are buried.

The main fragility I see is in the samples/ directory organization. The MANIFEST.md mislabels RECS 1993 microdata as "RECS 2001" — a fact I only caught because the sample size (7,111) matched the 1993 survey, not 2001 (4,822). If the MANIFEST can be wrong about something that fundamental, other mislabelings may exist. The samples are also not under version control (gitignored), which is correct per the charter but means the link between data files and their source documents is entirely editorial — if a sample file is deleted or renamed, the facts that cite it become unverifiable.

## On the work done

**What went well:** The Ramey verification was clean — pymupdf extracted the tables perfectly, and 27 of 28 values matched exactly. The one mismatch (1920s component breakdown using the 1965 column instead of the 1920s column) was exactly the kind of error the verification lesson predicts: the "famous" modern reference point (1965 AHTUS data) displaced the obscure historical data (Wilson 1920s study). The Statistical Abstract OCR via the local OCR host worked well once I found the right page — the chandra-ocr-2-mlx model produced clean table extractions at 400 DPI.

**What was awkward:** The EIA website is nearly impossible to scrape programmatically — every XLS download returns an HTML redirect page, and the data tables are JS-rendered. I had to rely on indirect verification (EIA overview page text, search results citing EIA data) for the RECS 2009 and 2015 heating fuel and AC values. The 1978 RECS data (first survey) appears to be completely inaccessible online. The 1990s cable TV "60% in 1992" figure could not be verified against its cited NCTA source — it appears to originate from Wikipedia, not the NCTA timeline PDF.

**What I'm confident in:** The Ramey corrections, the food price corrections (verified against OCR'd Table 530), the CEX expenditure shares (all 10 verified by subagent against source PDFs), the infant mortality arc (verified against NCHS Table 11), and the 1950s car price upgrade (Statistical Abstract Table 615 wholesale data is a solid primary source).

**What I'd want a second pair of eyes on:** The 1970s vehicle ownership income breakdown correction (47.3%→38.3%) was verified by a subagent against the CEX 1972-73 integrated PDF, but the specific table cell references should be double-checked. The 1990s cable TV fact still displays "~60%" despite acknowledging in the notes that it can't be verified — the honest choice might be to render the gap instead, but I kept the value because it's consistent with the trend and widely cited.

## Gaps to flag

- **MANIFEST.md mislabels RECS 1993 as RECS 2001** (`samples/MANIFEST.md:152-153`). The file2_asc.txt through file12_asc.txt are RECS 1993 data (7,111 sample), not RECS 2001 (4,822 sample). Other mislabelings may exist — the MANIFEST should be audited.

- **1990s cable TV "~60%" is unverified against its cited source** (`data/us/1990s.toml:188`). The NCTA Cable History Timeline PDF does not state a 60% figure for 1992. The value is consistent with the trend (50.5% in 1988 → 66.7M customers in 1999) but the primary source doesn't confirm it. Consider rendering the gap or finding a different primary source.

- **1978 RECS AC data is inaccessible** (`data/us/1970s.toml`). The 23% central AC figure for 1978 could not be directly verified — the first RECS survey data doesn't appear to be available online. The value is plausible (between 18.8% total AC in 1960 and 68% in 1993) but unverified.

- **EIA website blocks programmatic access** — all XLS downloads from eia.gov return HTML redirect pages. This makes ongoing verification of RECS data difficult. The Wayback Machine sometimes works but is unreliable for XLS files. Consider downloading and storing key EIA tables in samples/ for future verification.

- **The `nchs-nvss` source title says "Life Tables" but now also covers infant mortality** (`data/sources.toml:423`). The title should be updated to reflect both data series, or a separate source should be registered for the infant mortality table.

- **2000s housing expenditure share rounds to 32.3% but fact says 32.2%** (`data/us/2000s.toml:55`). Computed value is 32.27% — 0.07pp within rounding noise, but technically should round to 32.3%. Not corrected because the difference is within the source's dollar-rounded precision.
