## On the project

vitrine’s mechanical provenance gate keeps the renderer honest in a way most static-site generators are not. The constraint that every rendered mark must resolve to a curated fact (`data-fact-id` coverage) is the project’s real safety net — it caught me immediately when I changed corridor chart links from cross-page anchors to in-page overlays, because the old deep-link test assumed every `#` href pointed at a room file.

The template system is now at the edge of what inline Jinja2 strings can comfortably express. The popup-placard change required a shared macro template (`macros`) imported by room/corridor/pair templates, plus a hidden overlay deck on corridor pages. This works, but it’s a sign that the renderer is accumulating surface-specific complexity that may eventually want real template files rather than string constants. For now the DictLoader keeps everything in one place, which is fine.

## On the work done

**What went well:** The CSS-only overlay behaves exactly as the plan specified: click a glyph or chart point, the URL hash changes to `#fact-id--modal`, the page does not scroll, and closing the overlay clears the hash. The visual check confirmed a centered ivory card on a dimmed backdrop, with the close affordance floating at the top-right.

The no-JS rule held. The overlay uses `:target` + `position: fixed`; the inline placards remain for direct deep links and print fallbacks. I kept the walkthrough using room-placard links for now rather than forcing overlays onto a page that doesn’t render them.

**What was awkward:** Jinja2’s “extends must be first” rule meant I could not prepend macro definitions to templates that inherit `base`. I had to introduce a `macros` template and `{% from "macros" import ... %}` in each child. That’s standard Jinja2, but it added a touch of plumbing.

The local HTTP server aggressively cached the rebuilt `_site`, so several screenshots looked unchanged until I restarted the server. In production this is a non-issue because the files are static; during development it was a reminder that Playwright + a simple `http.server` retains pages.

**What I’m confident in:** The no-scroll popup interaction, the hitbox change, the collapsible panels, the axis rounding, the label-collision fix, the wording/label changes, and the decade disambiguation. The updated test correctly distinguishes cross-page deep links from in-page overlay anchors.

**What I’d want a second pair of eyes on:** The corridor overlay deck currently renders a hidden placard for every fact referenced on the page. This is correct for the no-JS popup model, but it inflates HTML size. I did not measure the impact on build output size; if it becomes significant, a future optimization could lazily render overlays or keep them in a shared deck. The test suite still passes and mark-coverage is green, so functionally it’s sound.
